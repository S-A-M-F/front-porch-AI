# Rawhide — What's New (Nightlies)

These notes feed the in-app "Update Available" dialog for Rawhide / cutting-edge builds.

## Recent improvements

- 📥 **Bulk-import Backyard AI characters** — the "Import BYAF" picker now lets you select **multiple `.byaf` files at once** (just like bulk-importing V2 PNG cards). A single file still shows the full preview; picking several imports them all with a progress bar, a per-batch "also import chat history" option, and a count of how many succeeded/failed.

- 🛠️ **Configure & launch your backend right from the Character Creator** — the Setup step now has dedicated KoboldCpp (local) and Pseudo-Remote sections with `.kcpps` preset + model selectors, a live status dot (Stopped → Starting… → Loading model… → Ready), a Start/Stop Backend button, and a foldable "Extra Settings" panel (GPU layers, context size, KV quantization, auto-configure). Remote (API) + oMLX share one searchable model picker. No more leaving the creator to start or tune your backend.

- 🎭 **AI Character Creator fully restored — it generates and saves real characters again** — At some point the creator had been quietly broken in two ways. On the surface, its setup screens were stripped down to bare boxes. Underneath it was worse: "generating" a character actually just produced a hardcoded placeholder (no AI was ever called), and the final Save step never saved anything. Everything is back and working: **Quick**, **Guided**, and **Automated** setup with all their options — the appearance builder, archetype quick-start presets, personality/backstory/NSFW trait chips, world-lore URL + file attachment, and the "magic wand" that writes a description for you. The **Realism** screen (seed a character's starting bond/trust/mood/needs) is wired up again, and the **Review** screen is back with avatar regenerate/crop, an editable card, and lorebook cherry-pick. Real AI generation and saving work end-to-end once more, with the original look and accent colors.

- 📁 **Subfolders no longer create phantom duplicate characters** — If you had a folder with characters, made a subfolder, and dragged a character into it, a duplicate copy used to appear back on the parent-folder screen — and deleting that "duplicate" would delete your real character too. Fixed: a character now lives only where you put it, with no phantom copy to accidentally delete. Large libraries also reload a little faster when you organize folders.

- 🛠️ **macOS release packaging fixed for v0.9.9.1 (and future)** — duplicate copy logic in the release workflow for kokoro_tts/piper PyInstaller bundles under Resources/piper caused a nested directory + "kokoro_tts/kokoro_tts: Not a directory" error during pkgbuild/productsign. Removed the redundant copy block (beta + nightly were already clean). Re-tagged v0.9.9.1 with the fix + full release notes from README.

- 🛠️ **KoboldCpp model picker in AI Character Creator now works + remote/oMLX unified** — the "Backend & Model Setup" step was always showing "No local models found" for Kobold (Local) even when .gguf files were present, and switching backends didn't update the model section. The scan was defined but never wired from the page or chip handlers, and no rebuild was triggered. Now scans on open + on Kobold select, forces UI refresh, preselects last-used, uses a scrollable list, disables reload until a model is picked, and shows load status. Remote (API) + oMLX now use the *exact same* direct inline model list UI (no more jarring fake dropdown field + separate dialog). "Browse / search all…" still available for huge remote lists. Model selection in the wizard is finally consistent and usable.

- 💾 **Manual Needs Reprocess now survives character message regen** — reprocessing needs updated the chips but not the frozen `realism_state` snapshot, so regenerating the character reply (new swipe) could leave the old swipe's manual corrections orphaned or inconsistent when you swiped back. Manual reprocess now syncs the needs vector into `realism_state`, regen explicitly preserves prior swipe metadata, and live sidebar state restores from the active swipe after regen.

- 🔄 **Manual Needs Reprocess no longer gets blank len=0 model responses** — thinking models on local KoboldCPP were hitting premature EOS during the `<think>` prefill, so the impact eval stream returned nothing and reprocess failed even though your critique was fine. Eval calls now use the same thinking-model safeguards as fact extraction (`ban_eos_token`), wait for Kobold to finish any in-flight work first, and automatically retry once on empty streams. Think-only outputs (no JSON after strip) are treated as failures so the reprocess retry path can recover. Original deltas still stay safe on failure.

- 🐛 **Realism deltas (Bond/Trust/Lust) fixed — "stop string" truncation was eating the JSON** — The eval engine was using stopSequences: ['}\n', '}\r\n', '```'] for all realism/impact calls. Because the relationship (and oneShot) prompts include free-text "xxx_reason" fields, a generated reason containing "}\n" (or the model emitting compact JSON without a newline after the final }) would cause the backend stop to fire in the middle of the object. Result: truncated raw, extractJsonInt for relationship_delta / trust_delta / arousal_delta returned null → deltas 0 or missing in pending/chips. The separate emotional_state call (simpler prompt, no long reasons) often succeeded, so "emotion will still process" while deltas were very hit-or-miss (periodically, depending on what the model put in the reason text). Fixed by removing the fragile stops (now [], like the other extracted eval leaves), relying on the strong "ONLY the JSON" instruction + temp 0.1 + regex extraction. Also switched trimStop off for the path. The previous "make them sequential" change helps with contention but the stop strings were the actual reported "stop string error". 1:1/group parity, analyze clean.
- 🛡️ **Manual Needs Reprocess is now safe and reliable** — failures (empty model response / len=0 / bad parse) no longer wipe chips, rewind live needs state, or break future action capture. Button gated to last + usable needs + !isGenerating. Non-destructive: pre-op snapshot; rollback on fail (historical: metadata on target msg only; live scalars/_groupRealism untouched). Commit/live only for last. Prompt via evaluate + group dance for speaker personality. Pre stash + revert (reasons preserved). Group sid + dance. 1:1/group exact. 0 new god void _; strengthened tests + stops; analyze clean.

After moving image gen settings ownership into the studio, the DT model list (via gRPC Echo hack) sometimes came back empty, causing only a text field to appear. Fixed by relaxing filters on both Python CLI and Dart sides (bare names + paths now survive), ensuring the DropdownButtonFormField is always the primary control (with current persisted model as fallback item), adding refresh button next to label, keyed FormField for reliable selection, and clean "Test to list models." message when empty. Selection is sent at generate time. Matches the pre-move UX where the dropdown "just worked". Test (or refresh) populates real choices from Draw Things.

- 🧬 **Character Evolution now fires on the schedule you actually set** — the "Evolve every N messages" slider (and `evolutionInterval` setting) was previously dead code; evolution always used the Auto-update persona interval (and a shared counter) regardless of what you picked or whether persona extraction was even enabled. Fixed with dedicated per-feature god-owned counters (`_userMessagesSinceLast*`) inside the existing periodic coordinator thins. Evolution now triggers exactly on its configured cadence (independent of facts), even in groups (per-utterance timing preserved), after chat switches/reloads, Director mode, etc. All ~15+ reset/zero hygiene sites + both startNew paths + "incomplete zeroing of secondary config... now complete" updated for the new evolution cadence counter. 0 new god private _ methods (count stayed exactly 15), 1:1/group parity for timing/target/counts/layering, analyze clean. The evolution slider finally does what it says.

- 🐛 **Fixed Realism Engine delta chips (Bond / Trust / Lust) — batch one-shot director slice now fully wired** — The collection path for the "fire the 5 mains then one director pass" (to reduce remote-API roundtrips on nano-gpt etc. when the per-char Director/Verifier is enabled) was clearing the pending batch items inside finalize before god could pull them for verifyBatch + apply. As a result the post-main parse/apply that feeds bond_delta / trust_delta / arousal_delta (Lust) into pending metadata never ran in the default 4-eval (!oneShot) path; only the later emotion_label synthesis survived, so the chip row showed only emotion (and needs). Fixed by: removing the premature clears + wouldVerify short-circuit from finalize (the real cheap fast-path lives inside verifyBatch), making applyBatchResults async + populating relationship (via _parseAndApply), emotional (scalars + arousal pending), and narrative (fixation + autonomous objective) side-effects from the (corrected or original) eff text for both send and regen paths (regen now mirrors the collected block), wiring the actual jsonDecode for the combined director response in verifyBatch so corrections are applied, and deduping the extraction logic into focused leaf helpers (direct paths now delegate too). 0 new god private _ methods (void _ count stayed 15), 1:1/group/oneShot parity, analyze clean, dart fix nothing, no new warnings. This is the completion of the prior "next slice" batch work so the system actually delivers the intended deltas/chips + one-director optimization.

- 🐛 **Prompt assembly state data cleaned up (targeted extraction) + realism eval speed for common case** — The part of prompt assembly that turns live internal state (needs x/100, bond/trust, emotion, time, arousal, spatial stance, etc.) into text the model sees was previously a loose concatenation of 8 differently-styled sub-blocks inside the god file, with needs using long prose instead of explicit numbers. This was the root of models "not seeing" or "failing to collate" the tracked scores into behavior. A new small `RealismStateInjection` composer (following the existing prompt_injection/ extraction pattern) now owns the presentation: one clear `[Speaker Internal State]` block with an explicit numeric metrics section first (all 7 needs as `Hunger: 42/100 (status)`, relationship scores, emotion, time, arousal, etc.), followed by the guidance text, ended by a strong collation instruction. The messy 8-line concat + individual thin wrappers were deleted from chat_service. Responses should now more reliably reflect the actual current internal numbers. This is incremental, safe cleanup of the "data sent to the model" layer (full top-to-bottom PromptAssembler extraction remains future work to avoid risk to quality).

  Additionally, the pre-response "5 realism evals" (relationship/emotional/physical/narrative/one-shot) now short-circuit expensive preState captures + bundle assembly entirely when the per-character "Director/Verifier" (realismVerificationEnabled) flag is off (the default). This restores original speed for the common "Realism Engine processing" phase on remote APIs (e.g. nano-gpt) where extra work or LLM critique passes were adding noticeable latency (often feeling ~2x). When you opt-in to the Verifier/Director authority for a character (higher fidelity reviewed deltas), the full review loops still run as before. Check the character editor / group settings "Director/Verifier" section if the processing phase feels slower than expected.

- 📁 **Folder previews on the home screen** — folders now show a 2×2 thumbnail of the first few characters inside them instead of a plain folder icon, so you can tell at a glance what's in each one. The preview scales with your grid size; empty folders keep the classic folder icon.

- **Realism 5-evals + Director batch "one shot" slice** — To address the "5 evals feel twice as slow on remote API (nano-gpt)" report, the pre-response realism evals now support the user's proposed flow: fire the mains (keep the parallel Future.wait for the 4), collect raws+full context in the leaf (evaluate* now short-circuit per-eval _verifyAndApply when batch active via beginCollect), then after the mains the god does the single _realismVerifier.verifyBatch using the collected (via getCollectedForBatch) and applies the (corrected) results (via applyBatchResults). The verifier has verifyBatch + _buildBatchCritiquePrompt for the combined one LLM critique pass on the whole set (local rules first, then at most one fire for corrections across all 5/4). This cuts verifier roundtrips from up to 5 to 1 when the per-char flag is on. The fast-path guard (flag off, default) keeps it free of extra work/captures. begin/finalize hooks + collection in the 4 evaluate paths + god wiring in the parallel blocks (send + group speaker). No new god private _ methods. Parity preserved. (Physical delegated to time, oneShot left as single for this slice.) Full apply extraction and test expansion in next slices. This is the "next slice" to make the system work better for remote users with/without the Director feature.

- 🧠 **Needs Simulation refinements** — Positive scene impacts on needs can now be much larger (up to +100 instead of the old hard +30 cap per evaluation), so a single strong replenishing action (big meal, thorough bath, long rest, caring affection) can meaningfully restore a need. The prompt injection now surfaces up to the two most pressing low needs (instead of only the single lowest) and includes the mild step-4 "background sensation" descriptions again. This restores progressive early hints, so characters (including on slow-decaying needs like Comfort and Hygiene) are more likely to notice and act before levels become critical. The Manual Reprocess / Director path also benefits from the larger positive range.

- 🔄 **Manual Needs Reprocessing** — you can now forcefully reprocess the Needs deltas of the latest AI message using the Director/Verifier. A new "Manual Reprocess" button sits below the Needs chips. Clicking it opens a dialog where you can enter a custom critique (e.g., "The character ate a granola bar and an energy drink"). The Realism Engine will then use your explicit feedback to reconstruct the scene's impacts on the character's Needs vector with the specified intensity scale. Look for the "✓ Director corrected (manual)" pill on the message once done!

- 🐛 **Fixed missing Needs reactions from characters** — Needs levels (hunger, comfort, bladder, etc.) were no longer impacting the character's internal thoughts and roleplay due to a regression where the descriptive text prompts were accidentally abbreviated in the code. We've restored the full injection prompts so characters will now appropriately complain or struggle when their needs hit noticeable or critical levels.

- 📋 **Group settings now has a dedicated "Needs" tab** — the needs simulation master toggle, per-character baseline sliders (Hunger, Bladder, Energy, Social, Fun, Hygiene, Comfort), and "Enjoys low hygiene" have all been moved from "Realism & Needs" into their own tab. The Director/Verifier controls (Max reprocesses, Strictness, Director authority) are now gated behind the Director/Verifier toggle so they don't clutter the UI when disabled. Also fixed: the character editor's needs baseline sliders were previously hardcoded to 80 with no-op callbacks — they now properly save your settings.

- 🎛️ **Group settings Realism tab now has editable realism baselines** — renamed from "Realism & Needs" (needs moved to its own tab). Group-level Time & Day selector (time of day + day count) added above per-character cards. Per-character cards now include editable sliders for Short-Term Bond (-300..300), Long-Term Bond (-300..300), and Trust Level (-100..100), plus inline fields for Starting Emotion (text + intensity dropdown). Changes persist to the group's baseline seed and per-member defaults so new chats start with your settings.

- 🎭 **Fork a 1:1 into a group with custom entrances for each new arrival** — "Fork to Group Chat" is now a quick step-by-step wizard instead of one cramped popup. Pick who joins (drag to set the order they enter), name the group and set the scene, then give *each* newcomer an optional entrance: **Opening line** (your text is used verbatim as their entrance, no AI) or **Direction** (the AI writes their entrance in-voice from your notes) — or leave it blank to just slot them into the turn order. Each entrance is delivered as a one-off turn; in Round Robin the conversation then resumes with whoever naturally follows the last new arrival, so the break-in doesn't scramble the rotation.

A new export button is available next to the import button in the User Personas tab. Clicking it lets you export all your personas at once into a single JSON file, or pick a specific persona to export. Personas' learned facts are safely embedded in the payload so you don't lose them when backing up or moving to another machine!
