# Rawhide — What's New (Nightlies)

These notes feed the in-app "Update Available" dialog for Rawhide / cutting-edge builds.

## Recent improvements

- 🧭 **Group Dynamics relationship sliders — now perfectly aligned with the Realism Engine + helpful tooltips** — The pre-seeding sliders (available for groups of 4 or fewer) now operate on the exact same -300..+300 raw scale and use the identical tier names ("Soulbound", "Deep Bond", "Familiar", "Acquaintance", "Estranged", "Broken", "Nemesis"...) as the Long-Term Bond controls in the 1:1 manual creator and the runtime _longTermTierName / prompt injection logic. Added clear tooltips on the numeric values, tier labels, and a new info icon in the step header explaining what these hidden scores do, how they persist in Group Cards, and why they only matter for small groups. Breakpoints and stepping are now 1:1 consistent with the rest of the Realism system.

- 📦 **Group Card PNG export/import now fully round-trips `defaultMemberRealismState`** — Pre-seeded Group Dynamics relationships, per-member needs vectors, "Enjoys low hygiene" flags, and the rest of the rich definition-level starting state now survive export as a `.group.png` and are restored on import. This completes the v30+ schema contract (the DB column had long claimed Group Card support; the interchange layer now delivers it). Old cards remain fully compatible. New groups created with the wizard (including hidden intra-group feelings) will now produce faithful portable Group Cards that recreate the exact intended starting conditions elsewhere or after a split-to-solo.

- ✨ **Group Chat Creator — now fully consistent with the 1:1 character creators** — Major polish pass: the group creator now uses the exact same top-bar step indicator + linear Next/Back flow as the manual character creator. Fixed several footguns — Time of Day and Day Count are now group-level global settings (no more different characters on different days in the same chat), removed duplicate per-character "Enable Realism Engine" toggles, added a proper global Needs Simulation toggle nested under the Realism master, styled the Chaos card nicely, and ensured "Enjoys low hygiene" only appears under Optional Features when the group-level Needs is enabled. The Realism step is now safe and first-class.

- ✨ **Group Chat Creator — complete first-class rewrite** — The old cramped "select 2+ characters → bottom bar" dialog is gone forever. Group creation is now a persistent, attractive sidebar button (just like the AI / Manual character creators). It launches a beautiful menu-driven page with a rich roster builder (searchable character browser, drag-to-reorder, per-member voices), full support for group lorebooks, linked worlds, inherit toggle, per-character system prompt overrides, Director Mode, Chaos (with NSFW), and complete per-member Realism/Needs seeding (baseline + defaultMember state for perfect Group Card fidelity and split-to-solo). Create & immediately enter the chat. Everything you can do in Group Settings is now available at birth — beautiful, powerful, and on par with the 1:1 experience.

- 🧹 **Group Settings dialog — massive UX cleanup** — The Group Settings dialog had accumulated a ridiculous number of save buttons (per-tab bars + sectional saves + global "Save All Changes" + Done). This has been overhauled: edits now apply live to the running group, per-tab "Save Changes" bars have been removed across all tabs, and there is now one clear "Save" action in the main footer for persistence. The particularly messy Lorebook & Worlds tab was a primary target of the cleanup.

- 🎭 **Group chat UI completely overhauled — no more bolted-on jank** — The group sidebar has been rebuilt from the ground up. Each member now has a clean, modern card: the active speaker gets the full rich 1:1-style realism view (separate Short-Term + Long-Term Bond rows with proper progress bars, full 7-need grid with critical highlighting, proper fixation card with lifespan + manual clear, etc.). Other members stay compact and scannable with emotion rings, tiny tier indicators, fixation tags, and mini needs urgency. App bar stacked avatars now show live emotion-colored rings + hover tooltips (name + emotion + fixation). The old cramped ListTile subtitle explosions, hidden "info" popups for needs, and 10pt italic fixation text are gone forever. Objectives are now managed in a proper per-character group dialog without global state swapping hacks. Everything feels designed for groups.

- 🛡️ **Your years of character chats are now safe from silent schema problems** — Long-time users (especially those with large databases full of group chats, objectives, Chaos/Realism/Needs history) will no longer hit "no such column" crashes or broken group features after app updates. The database layer now performs an automatic, non-destructive repair on every launch: it inspects the real columns with PRAGMA, adds any that a past migration silently missed (chat_id scoping, all the v30-v32 group columns for baseline realism, Chaos toggles, lore/world scoping, per-character system prompts, etc.), and — the first time it actually changes anything — creates a timestamped backup copy of your .db right next to the original. You can always rename the backup back if you ever want to roll back. Deleting or resetting your database is never required or suggested. Group chat parity features (per-character objectives, Chaos sidebar, full Group Card roundtrips, etc.) now work reliably even on the oldest surviving user databases.

- 🛠️ **Group Chat creator now supports per-character baseline realism + group-scoped system prompts at creation time** — The Create Group dialog has been overhauled with full per-character editors for baseline Bond/Trust/Emotion/Arousal/Needs/Fixations (matching the live realism engine) and dedicated group-scoped system prompt text fields for each member. These seed the initial state exactly like the Group Settings dialog, so groups start with the intended personality overrides and realism baselines instead of defaults. Both creator and settings paths now use the same v30-backed persistence for group_realism_state and per-character group prompt overrides.

- 🧠 **Group realism & needs are now properly stored in the database (no more hidden jank)** — Per-character Bond/Trust/Emotion/Fixation/Arousal/Relationships + full Needs simulation state for every member in a group chat now lives in the new `sessions.group_realism_state` and `groups.default_member_realism_state` columns (schema v30, clean break). This was the last piece needed for high-fidelity Group Cards: exported `.group.png` files now carry the current evolved state for all members. When you split a group member out to a solo character, they keep the relationships, needs, and fixation they developed inside the group. Import round-trips the state perfectly into the new group definition. The old `__group_state__` sentinel message system is fully removed.
