# Rawhide — What's New (Nightlies)

These notes feed the in-app "Update Available" dialog for Rawhide / cutting-edge builds.

## Recent improvements

- 🧠 **Director/Reviewer now reliably surfaces lingering thoughts (fixations) + needs chips even on corrected turns** — the new optional Realism Verification (Director) reprocess path for narrative (which drives fixation_topic / "CURRENT FIXATION" lingering intrusive thoughts) and needs_impact could drop or reformat the JSON fields the model originally emitted. The Director's critique prompt now explicitly reminds to preserve "fixation_topic" + "proposed_objective" (and needs delta keys + reason). Extraction for both now does a tolerant JSON object decode (after fence stripping) with regex fallback so corrections that reformat still yield the (possibly adjusted) values. Result: important emotional/possessive/autonomous turns now properly set visible lingering fixations instead of silently "none"; needs scene deltas (and decay chips) emit more reliably to bubbles + sidebar even when Director authority is on and strict. No extra eval cost.

- 🧠 **Needs prompt fully opened — model takes the wheel like bond/emotion/relationship** — stripped the remaining "only if unambiguous completed act", "no metaphors", "pure sex scenes must give neutral energy/hunger" etc. rules from the needs impact prompt. It is now open like the other realism evals: the model reads the actual written scene (detailed erotic physical/psychological narration counts) and outputs reasonable net deltas. Only broad code clamps (-30..+30 to catch crazy output) + the Director/Verifier (when you enable per-char verification + "Director authority on needs deltas") for scene faithfulness, exactly like the other realism calls. No more prompt-level handcuffs. Strength slider is your main swing-size control. This matches the post-simplification intent ("straight model deltas + optional Director, just clamps").

- 📦 **macOS in-app updater now works with .pkg releases + seamless transition for everyone** — primary releases are now signed+notarized .pkg (better install, Gatekeeper, sidecar layout). Added the one last unsigned/un-notarized shim .dmg (Front_Porch_AI_Nightly.dmg, Front_Porch_AI_MacOS.dmg, Front_Porch_AI.dmg) so *all* macOS users — whether they installed via old drag-from-DMG .app or the new .pkg — can still hit "Update" inside the app and get a smooth auto-update without manually downloading anything. Old clients keep working (they see the shim), new clients prefer the .pkg but fall back gracefully. The shims are the bridge only; future releases can drop them once everyone has moved over. 🎉

- 🐛 **Fixed red screen crash in chat after realism/needs/Director turns** — some users would see repeated "RangeError (length): Invalid value: Valid value range is empty: -1" big red errors in the message list (and the app feeling broken) right after turns that used the Realism Engine, Needs Simulation, or the new Director/Verifier. The chip row code that shows things like "Bond +2", "Trust +5", "✓ Director accepted", "🕵️ Director corrected", "Hunger +3", "Energy -2" etc. under AI messages had a fragile "add spacers then chop the last one" trick that assumed there would always be at least one chip when the indicator decided it had something to display. This broke when a message only had needs deltas (very common now with the improved needs path), or when all the deltas filtered to zero while the guard had still fired, or on history rebuilds for older messages once verifier metadata started landing. Fixed: safe spacing helper, proper guards, and we no longer render empty "classic" boxes when a turn only moved needs. The Director and needs chips now work reliably for everyone. Update your Rawhide build to get the fix — no more surprise crashes from the realism system.

- 🛠️ **Needs delta strength slider clarified + Director no longer double-scales** — the "Needs delta strength: 5x" slider (in Optional Features when Needs + the "Director authority on needs deltas" toggle are active) had ambiguous wording. It now clearly states that the multiplier is instructed to the *model on the first needs-impact call* (and to the Director when authority is on) so they emit/correct deltas at the requested magnitude in one pass (normal -3 hygiene → ~-15 at 5x). The Director works in the scaled space. We removed the previous post-processing multiplication that would have taken a Director-corrected -15 and turned it into -75. What the model (or Director-corrected output) returns is applied directly. Matches the requested contract: "multiplier at first run; Director must not take multiplied delta and multiply it again."

- 🎚️ **Needs delta strength (1x–5x exponent) + trust delta robustness** — new 1x–5x control in character edit Details (right-click flow, same Optional Features container as Director/Verifier toggles+sliders). Integer multiplier (default 1x) is injected into the first-pass needs impact eval prompt *and* the Director (needs_impact) so the model and corrections emit at the user-requested magnitude (e.g. model hygiene -3 at 1x becomes -15 at 5x). The Director corrects in the scaled space; the returned numbers are applied as-is (no extra post-multiply on top of Director output). Per-card + per-member in groups. Directly addresses user requests for control over needs delta magnitude ("small lever to control how fast/slow"). Also: trust deltas from relationship evals are now reliably recorded in pending/metadata/attachment even at 0 when the eval produced a non-"none" reason (the previous nonzero-only guards were dropping them from chips/synthesis/logs on accepted verifier output). (Full gates + literal cd+abs+cat hygiene + "0 open" re-reads; 0 new god privs; void_=15; analyze clean on changed; manual smoke with authority+5x on complex scenes.)

"Director authority on needs deltas" toggle (Optional Features, off default) + getNeedsModelAuthorityEnabled cb + clean branch in needs impact: when on (with Verification), Director reloop corrections have authority and drive applied deltas/chips/sidebar/injections (effective from corrected); legacy off = straight model deltas only. - 🕵️ **Optional Realism Verification (Director/Verifier) thread** — per-character toggle in Optional Features (Details tab on right-click edit, plus full creator/edit pages and per-member in group flows). Ingests the *full* latent decision context the Realism Engine + Needs had (exact prompt + all injected blocks + pre-turn scalars + needs vector + recent messages + active speaker CharacterCard with personality/frontPorch + group + eval kind + raw output). - 🐛 **Fixed bond doubling for V2.5 card seeds on new 1:1 chats** — fresh imports and "New Chat" now correctly seed the author-intended short/long-term bond values from the card (e.g. 55 stays 55, not incorrectly doubled to 110). V2.5 cards + creator have always authored on the current ±300 scale; the legacy migration (*2 for old ±150 persisted sessions) was being erroneously applied to every card-seed path. Legacy session loads continue to auto-upgrade old data. Group paths were unaffected. 1:1/group parity and all reset hygiene preserved. (Also added central dispose guard protecting async post-dispose notifyListeners across the god file for prod stability + fewer "used after disposed" flakes in tests.)
- 🛠️ **Realism delta clamps unified (trust & arousal)** — trust_delta and arousal_delta limits are now identical whether using the default multi-call evals or the experimental One-Shot Eval setting. All per-turn clamp numbers (relationship/bond ±15, trust -200..+50, arousal ±25) are named constants that also feed the prompt text the model sees, so the "intended range" and the enforcement can never get out of sync again. Fixes a subtle inconsistency where one-shot vs. multi-call (and prompt vs. code) allowed different max swings on the same turn.

- 🏗️ **Major god-file modularization (Stages 1–7) landed on Rawhide** — the largest "god files" (chat_service.dart, storage_service, key UI pages, etc.) have been decomposed into focused, single-responsibility modules while preserving 100% behavioral parity for chats, groups, Realism Engine (bond/trust/emotion/arousal/fixation/needs), objectives, fact extraction, character evolution, RAG, prompt injection, and all creator flows. The full test suite is now reliably green (+1126 tests) as the new baseline. This was developed on an isolated branch and is being merged via PR (no history rewrite) to keep Rawhide as the primary development line. See [refactoring-guide.md](refactoring-guide.md) and the branch notes for details.

- 🎯 **Autonomous character objectives now reliably get subtasks** — when the AI (via realism evals) proposes a personal goal for a character ("proposed_objective"), it now properly auto-generates 3 concrete sequential tasks the character can work toward. This was previously unreliable (especially in groups) due to character targeting during per-speaker evals and fragile post-set lookup. User-created objectives (typed manually in the UI) correctly do *not* auto-generate tasks — the player remains in control and can use the Generate Tasks button when desired.

- 🧠 **Thinking/reasoning models now have breathing room for objectives & tasks** — Subtask generation (`generateObjectiveTasks`) and objective/task completion checks bumped from 600/1024 to 2000 max tokens. `<think>...</think>` stripping (now using the central robust helper that also handles unclosed tags) happens after the full stream so models can think at length before emitting the final numbered list or YES/NO. This was broken for most thinking models that reason more than a few hundred tokens. Autonomous proposals already had a generous 4000.

- 🔏 **Fixed macOS notarization** — nightly builds now pass Apple's notary service correctly. Python.framework bundles inside the AI sidecars are now signed as proper framework units (instead of file-by-file), resolving "signature of binary is invalid" and "not signed with valid Developer ID" errors.

- 🐛 **Realism, Needs & Group reliability** — Sidebar bond/trust values + chips now stay in sync; group per-speaker needs, scene rewards (fun/social/hygiene from activities), decay, and cards persist and display correctly with proper pre/post snapshots. Zero-delta chips suppressed, duplicate messages on load fixed, no more needs/fixation/relationship bleed on fresh starts, forks, or imports, and better 1:1 vs group parity for all post-gen effects.

- ✨ **Home screen & daily polish** — Refresh button next to multi-select for re-scanning external character imports. Broader packaging/CI wins for distinct Rawhide nightlies (no name collisions with stable) and many small stability/hygiene improvements.

- 🐛 **Fork to Group Chat member population** — Fixed the bug (originally contributed as PR #44 by @MisterLotto) where forking a 1:1 chat into a group would create a GroupChat with zero members in the decoupled `group_members` table. Both the initial fork path and "add character to existing group" now go through shared creation logic for the avatar copy + DB row. This was held during the god-file modularization to avoid divergence and is now landed on current Rawhide.
