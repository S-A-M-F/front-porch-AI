# Rawhide — What's New (Nightlies)

These notes feed the in-app "Update Available" dialog for Rawhide / cutting-edge builds.

## Recent improvements

- 🎚️ **Needs delta strength (1x–5x exponent) + trust delta robustness** — new 1x–5x control in character edit Details (right-click flow, same Optional Features container as Director/Verifier toggles+sliders). Integer multiplier (default 1x) is injected into the first-pass needs impact eval prompt *and* the Director (needs_impact) so the model and corrections emit at the user-requested magnitude (e.g. model hygiene -3 at 1x becomes -15 at 5x). Final deltas = raw × strength in the thin path (baseline + Director). Per-card + per-member in groups. Directly addresses user requests for control over needs delta magnitude ("small lever to control how fast/slow"). Also: trust deltas from relationship evals are now reliably recorded in pending/metadata/attachment even at 0 when the eval produced a non-"none" reason (the previous nonzero-only guards were dropping them from chips/synthesis/logs on accepted verifier output). (Full gates + literal cd+abs+cat hygiene + "0 open" re-reads; 0 new god privs; void_=15; analyze clean on changed; manual smoke with authority+5x on complex scenes.)

- 🕵️ **Director authority on needs deltas (simple model+Director path)** — per "strip out all the buffer afterglow shit... just straight up decay ticks and model deltas with the optional director review loop" + "way too complicated". "Director authority on needs deltas" toggle (Optional Features, off default) + getNeedsModelAuthorityEnabled cb + clean branch in needs impact: when on (with Verification), Director reloop corrections have authority and drive applied deltas/chips/sidebar/injections (effective from corrected); legacy off = straight model deltas only. No table/6 modifiers/afterglow/crash/buffers (fully expunged). Classic reloop strengthened to ensure critique changes deltas. (See .claude/changelog.md + review_file for gates/raw/0 open; realism-engine.md for technical details.)

- 🕵️ **Optional Realism Verification (Director/Verifier) thread** — per-character toggle in Optional Features (Details tab on right-click edit, plus full creator/edit pages and per-member in group flows). Ingests the *full* latent decision context the Realism Engine + Needs had (exact prompt + all injected blocks + pre-turn scalars + needs vector + recent messages + active speaker CharacterCard with personality/frontPorch + group + eval kind + raw output). Rule checks (using authoritative per-eval clamp limits) + optional reprocess (slider 1-5 max passes) with explicit reason + corrections (allowed to swing full clamp ranges). Strictness/leniency slider (1-5) tunes rule bar and reprocess prompt tone. Visuals reuse the exact existing Realism processing overlay ("🕵️ Verifying Realism output (pass X/Y)") and the realism/needs chip row in message bubbles ("✓ Director accepted" / "🕵️ Director corrected (N reprocesses)"). Off by default (zero cost/latency). 1:1 vs group + oneShot parity preserved. Strong-model users get higher-fidelity living deltas with visible director feedback. (Full leaf extraction, 0 new god void _, keep-sync hygiene, dedicated test, gates + manual smoke.)

- 🐛 **Fixed bond doubling for V2.5 card seeds on new 1:1 chats** — fresh imports and "New Chat" now correctly seed the author-intended short/long-term bond values from the card (e.g. 55 stays 55, not incorrectly doubled to 110). V2.5 cards + creator have always authored on the current ±300 scale; the legacy migration (*2 for old ±150 persisted sessions) was being erroneously applied to every card-seed path. Legacy session loads continue to auto-upgrade old data. Group paths were unaffected. 1:1/group parity and all reset hygiene preserved. (Also added central dispose guard protecting async post-dispose notifyListeners across the god file for prod stability + fewer "used after disposed" flakes in tests.)
- 🛠️ **Realism delta clamps unified (trust & arousal)** — trust_delta and arousal_delta limits are now identical whether using the default multi-call evals or the experimental One-Shot Eval setting. All per-turn clamp numbers (relationship/bond ±15, trust -200..+50, arousal ±25) are named constants that also feed the prompt text the model sees, so the "intended range" and the enforcement can never get out of sync again. Fixes a subtle inconsistency where one-shot vs. multi-call (and prompt vs. code) allowed different max swings on the same turn.

- 🏗️ **Major god-file modularization (Stages 1–7) landed on Rawhide** — the largest "god files" (chat_service.dart, storage_service, key UI pages, etc.) have been decomposed into focused, single-responsibility modules while preserving 100% behavioral parity for chats, groups, Realism Engine (bond/trust/emotion/arousal/fixation/needs), objectives, fact extraction, character evolution, RAG, prompt injection, and all creator flows. The full test suite is now reliably green (+1126 tests) as the new baseline. This was developed on an isolated branch and is being merged via PR (no history rewrite) to keep Rawhide as the primary development line. See [refactoring-guide.md](refactoring-guide.md) and the branch notes for details.

- 🎯 **Autonomous character objectives now reliably get subtasks** — when the AI (via realism evals) proposes a personal goal for a character ("proposed_objective"), it now properly auto-generates 3 concrete sequential tasks the character can work toward. This was previously unreliable (especially in groups) due to character targeting during per-speaker evals and fragile post-set lookup. User-created objectives (typed manually in the UI) correctly do *not* auto-generate tasks — the player remains in control and can use the Generate Tasks button when desired.

- 🧠 **Thinking/reasoning models now have breathing room for objectives & tasks** — Subtask generation (`generateObjectiveTasks`) and objective/task completion checks bumped from 600/1024 to 2000 max tokens. `<think>...</think>` stripping (now using the central robust helper that also handles unclosed tags) happens after the full stream so models can think at length before emitting the final numbered list or YES/NO. This was broken for most thinking models that reason more than a few hundred tokens. Autonomous proposals already had a generous 4000.

- 🔏 **Fixed macOS notarization** — nightly builds now pass Apple's notary service correctly. Python.framework bundles inside the AI sidecars are now signed as proper framework units (instead of file-by-file), resolving "signature of binary is invalid" and "not signed with valid Developer ID" errors.

- 🐛 **Realism, Needs & Group reliability** — Sidebar bond/trust values + chips now stay in sync; group per-speaker needs, scene rewards (fun/social/hygiene from activities), decay, and cards persist and display correctly with proper pre/post snapshots. Zero-delta chips suppressed, duplicate messages on load fixed, no more needs/fixation/relationship bleed on fresh starts, forks, or imports, and better 1:1 vs group parity for all post-gen effects.

- ✨ **Home screen & daily polish** — Refresh button next to multi-select for re-scanning external character imports. Broader packaging/CI wins for distinct Rawhide nightlies (no name collisions with stable) and many small stability/hygiene improvements.

- 🐛 **Fork to Group Chat member population** — Fixed the bug (originally contributed as PR #44 by @MisterLotto) where forking a 1:1 chat into a group would create a GroupChat with zero members in the decoupled `group_members` table. Both the initial fork path and "add character to existing group" now go through shared creation logic for the avatar copy + DB row. This was held during the god-file modularization to avoid divergence and is now landed on current Rawhide.
