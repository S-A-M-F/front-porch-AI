# Rawhide — What's New (Nightlies)

These notes feed the in-app "Update Available" dialog for Rawhide / cutting-edge builds.

## Recent improvements

- 🖥️ **Pseudo-Remote backend is now fully wired** — You can select it in Settings → Backend, configure the same .kcpps + exe you use for native Kobold, hit Start, and it launches the process while routing chat/Realism/Needs/reasoning through the OpenAI-compatible /v1 endpoint (the original goal of PR #33). The settings UI, start/stop, logs, and model switching all work for both managed backends. Light-mode color persistence foundation also landed (Dark Mode toggle now survives restarts and the 5 chat bubble colors have separate dark/light values).
- 🧹 **Major codebase hygiene pass** — Eliminated 150+ style, import, null-assertion, and generated-code lint warnings (from 206 down to ~50 remaining). Added proper `analyzer: exclude` rules for gRPC/FlatBuffer generated files and removed an invalid linter suppression. Bulk mechanical fixes via `dart fix` (braces, non-null asserts, Flutter deprecation migrations, etc.) plus removal of a few truly dead constants/methods. `flutter analyze` and IDEs are now far quieter; this is the first step of a phased plan to make the linter strict again without blocking development.
- 🖥️ **Pseudo-Remote backend** — A new backend option that runs KoboldCPP locally on your machine while exposing it through an OpenAI-compatible API. This gives you the strengths of local models (no API costs, full control) combined with the flexibility of remote-style prompting and generation. The integration is now complete and stable after the initial build issues.
- 🔄 **Auto-updater for Rawhide** — Nightly builds now correctly detect newer Rawhide releases and offer in-app updates on all platforms. The asset lookup, version comparison (date-based for `rawhide.YYYYMMDD.SHA`), and display strings were fixed so older nightlies will see and install newer ones.

- 🧠 **Needs Simulation — now a true first-class system** (like bond, trust, emotion, and fixation). The previous janky side-channel detectors have been replaced with a single, rich, model-driven structured evaluation that understands context, pleasure, embarrassment, relief, and erotic nuance. Characters now generate natural first-person internal-thought narrative memories for big need events (short-term "lingering thoughts") plus invisible long-term RAG recall so they can remember high-impact pleasure or humiliation events many turns later when contextually relevant. 

  Recent refinements add proactive pre-response Needs pressure (with OOC guidance for urgent or high states) so characters naturally express comfort, pleasure, or distress in the moment. High positive states (especially Fun, Comfort, Social) now generate appropriate OOC context. The "Enjoys low hygiene" reversal is supported — characters who revel in being filthy get rewarded for gross acts while the stat still tracks actual dirtiness. Regen/swipe now properly carries rich Needs deltas and chips. All needs that move (including small natural decay) now produce visible chips with reasons every turn.
