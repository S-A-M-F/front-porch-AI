# Rawhide — What's New (Nightlies)

These notes feed the in-app "Update Available" dialog for Rawhide / cutting-edge builds.

## Recent improvements

- 🔄 **Manual Needs Reprocessing** — you can now forcefully reprocess the Needs deltas of the latest AI message using the Director/Verifier. A new "Manual Reprocess" button sits below the Needs chips. Clicking it opens a dialog where you can enter a custom critique (e.g., "The character ate a granola bar and an energy drink"). The Realism Engine will then use your explicit feedback to reconstruct the scene's impacts on the character's Needs vector with the specified intensity scale. Look for the "✓ Director corrected (manual)" pill on the message once done!

- 🐛 **Fixed missing Needs reactions from characters** — Needs levels (hunger, comfort, bladder, etc.) were no longer impacting the character's internal thoughts and roleplay due to a regression where the descriptive text prompts were accidentally abbreviated in the code. We've restored the full injection prompts so characters will now appropriately complain or struggle when their needs hit noticeable or critical levels.

- 📋 **Group settings now has a dedicated "Needs" tab** — the needs simulation master toggle, per-character baseline sliders (Hunger, Bladder, Energy, Social, Fun, Hygiene, Comfort), and "Enjoys low hygiene" have all been moved from "Realism & Needs" into their own tab. The Director/Verifier controls (Max reprocesses, Strictness, Director authority) are now gated behind the Director/Verifier toggle so they don't clutter the UI when disabled. Also fixed: the character editor's needs baseline sliders were previously hardcoded to 80 with no-op callbacks — they now properly save your settings.

- 🎛️ **Group settings Realism tab now has editable realism baselines** — renamed from "Realism & Needs" (needs moved to its own tab). Group-level Time & Day selector (time of day + day count) added above per-character cards. Per-character cards now include editable sliders for Short-Term Bond (-300..300), Long-Term Bond (-300..300), and Trust Level (-100..100), plus inline fields for Starting Emotion (text + intensity dropdown). Changes persist to the group's baseline seed and per-member defaults so new chats start with your settings.

- 🎭 **Fork a 1:1 into a group with custom entrances for each new arrival** — "Fork to Group Chat" is now a quick step-by-step wizard instead of one cramped popup. Pick who joins (drag to set the order they enter), name the group and set the scene, then give *each* newcomer an optional entrance: **Opening line** (your text is used verbatim as their entrance, no AI) or **Direction** (the AI writes their entrance in-voice from your notes) — or leave it blank to just slot them into the turn order. Each entrance is delivered as a one-off turn; in Round Robin the conversation then resumes with whoever naturally follows the last new arrival, so the break-in doesn't scramble the rotation.

- 🧬 **Character evolution now robustly parses LLM output from remote/thinking models** — the evolution feature (manual or periodic) was frequently failing with "JSON parse attempt failed" / "Missing fields" even when the model had done the rewrite work. Root causes: greedy `\{.*\}` would latch onto stray `{` from `{{char}}` macros inside the personality text the model echoed or wrote; the "truncated recovery" was a naive append that almost never worked; only `<think>` tags were stripped (many remote backends just emit plain English reasoning first); and the "return ONLY JSON" instruction was buried at the end of a 48k-char prompt. Fixed with key-anchored extraction (prefer the object containing `"personality"` / `"scenario"`), a proper brace-counting repair (ported from story pipeline), safe casts, preview logging on failures, and a much stronger "your *entire* response must be exactly one JSON object" instruction at the prompt head + reinforced before Response:. **Further hardened**: when the model completely ignores the wrapper and just emits the rewritten personality as free prose (the "{{char}} is sharp, playful..." style with no JSON or labels at all), we now detect it, strip the leading reasoning, salvage the descriptive block, reuse the prior scenario for safety, log "⚠ Model emitted raw prose...", and still persist + apply the [Character Growth] layer. Labeled `PERSONALITY:\n...\n\nSCENARIO:\n...` sections are also supported as an explicit escape hatch in the prompt. The exact logs you showed (16k+ char "JSON candidate" that was actually the personality text, char 2 pointing at {{char}}) are now handled via the prose salvage path. Manual re-trigger will pick up the growth for affected characters.

- 🕵️ **Stronger dedicated test surface for Director/Verifier (realism_verification)** — the optional per-character Realism Verification / Director review loop now has a significantly improved test factory (`createTestRealismVerification`) that makes it trivial to supply a full `CharacterCard` with `FrontPorchExtensions` realism verification flags + `realismNeedsDirectorAuthority`. Added deterministic exercising tests for the key responsibilities that were recently hardened (fixation_topic nulled when unsupported + strict, preservation of "proposed_objective" and "fixation_topic" through reprocess per structHint, needs delta + reason keys surviving corrections, critique prompt containing the "Preserve full shape..." and needs-keys hints, cancel mid-reprocess, strictness effects, clamp respect on deltas, rich latent bundle, 1:1 vs group cbs, authority card in bundle). Suite now at 25 focused bodies. aug tests (needs_impact, realism_evals, etc.) only received qualified passive notes. This gives reliable regression coverage for the Director without having to drive the full god file.

- 🧠 **Director/Reviewer now reliably surfaces lingering thoughts (fixations) + needs chips even on corrected turns** — the new optional Realism Verification (Director) reprocess path for narrative (which drives fixation_topic / "CURRENT FIXATION" lingering intrusive thoughts) and needs_impact could drop or reformat the JSON fields the model originally emitted. The Director's critique prompt now explicitly reminds to preserve "fixation_topic" + "proposed_objective" (and needs delta keys + reason). Extraction for both now does a tolerant JSON object decode (after fence stripping) with regex fallback so corrections that reformat still yield the (possibly adjusted) values. Result: important emotional/possessive/autonomous turns now properly set visible lingering fixations instead of silently "none"; needs scene deltas (and decay chips) emit more reliably to bubbles + sidebar even when Director authority is on and strict. No extra eval cost.

- 🧠 **Needs prompt fully opened — model takes the wheel like bond/emotion/relationship** — stripped the remaining "only if unambiguous completed act", "no metaphors", "pure sex scenes must give neutral energy/hunger" etc. rules from the needs impact prompt. It is now open like the other realism evals: the model reads the actual written scene (detailed erotic physical/psychological narration counts) and outputs reasonable net deltas. Only broad code clamps (-30..+30 to catch crazy output) + the Director/Verifier (when you enable per-char verification + "Director authority on needs deltas") for scene faithfulness, exactly like the other realism calls. No more prompt-level handcuffs. Strength slider is your main swing-size control. This matches the post-simplification intent ("straight model deltas + optional Director, just clamps").

- 📦 **macOS in-app updater now works with .pkg releases + seamless transition for everyone** — primary releases are now signed+notarized .pkg (better install, Gatekeeper, sidecar layout). Added the one last unsigned/un-notarized shim .dmg (Front_Porch_AI_Nightly.dmg, Front_Porch_AI_MacOS.dmg, Front_Porch_AI.dmg) so *all* macOS users — whether they installed via old drag-from-DMG .app or the new .pkg — can still hit "Update" inside the app and get a smooth auto-update without manually downloading anything. Old clients keep working (they see the shim), new clients prefer the .pkg but fall back gracefully. The shims are the bridge only; future releases can drop them once everyone has moved over. 🎉

- 🐛 **Fixed red screen crash in chat after realism/needs/Director turns** — some users would see repeated "RangeError (length): Invalid value: Valid value range is empty: -1" big red errors in the message list (and the app feeling broken) right after turns that used the Realism Engine, Needs Simulation, or the new Director/Verifier. The chip row code that shows things like "Bond +2", "Trust +5", "✓ Director accepted", "🕵️ Director corrected", "Hunger +3", "Energy -2" etc. under AI messages had a fragile "add spacers then chop the last one" trick that assumed there would always be at least one chip when the indicator decided it had something to display. This broke when a message only had needs deltas (very common now with the improved needs path), or when all the deltas filtered to zero while the guard had still fired, or on history rebuilds for older messages once verifier metadata started landing. Fixed: safe spacing helper, proper guards, and we no longer render empty "classic" boxes when a turn only moved needs. The Director and needs chips now work reliably for everyone. Update your Rawhide build to get the fix — no more surprise crashes from the realism system.

- 🛠️ **Needs delta strength slider clarified + Director no longer double-scales** — the "Needs delta strength: 5x" slider (in Optional Features when Needs + the "Director authority on needs deltas" toggle are active) had ambiguous wording. It now clearly states that the multiplier is instructed to the *model on the first needs-impact call* (and to the Director when authority is on) so they emit/correct deltas at the requested magnitude in one pass (normal -3 hygiene → ~-15 at 5x). The Director works in the scaled space. We removed the previous post-processing multiplication that would have taken a Director-corrected -15 and turned it into -75. What the model (or Director-corrected output) returns is applied directly. Matches the requested contract: "multiplier at first run; Director must not take multiplied delta and multiply it again."

The Director corrects in the scaled space; the returned numbers are applied as-is (no extra post-multiply on top of Director output). Per-card + per-member in groups. Directly addresses user requests for control over needs delta magnitude ("small lever to control how fast/slow"). Also: trust deltas from relationship evals are now reliably recorded in pending/metadata/attachment even at 0 when the eval produced a non-"none" reason (the previous nonzero-only guards were dropping them from chips/synthesis/logs on accepted verifier output). - 🐛 **Fixed bond doubling for V2.5 card seeds on new 1:1 chats** — fresh imports and "New Chat" now correctly seed the author-intended short/long-term bond values from the card (e.g. 55 stays 55, not incorrectly doubled to 110). V2.5 cards + creator have always authored on the current ±300 scale; the legacy migration (*2 for old ±150 persisted sessions) was being erroneously applied to every card-seed path. - 🛠️ **Realism delta clamps unified (trust & arousal)** — trust_delta and arousal_delta limits are now identical whether using the default multi-call evals or the experimental One-Shot Eval setting. All per-turn clamp numbers (relationship/bond ±15, trust -200..+50, arousal ±25) are named constants that also feed the prompt text the model sees, so the "intended range" and the enforcement can never get out of sync again. Fixes a subtle inconsistency where one-shot vs. multi-call (and prompt vs. code) allowed different max swings on the same turn.

- 🎯 **Autonomous character objectives now reliably get subtasks** — when the AI (via realism evals) proposes a personal goal for a character ("proposed_objective"), it now properly auto-generates 3 concrete sequential tasks the character can work toward. User-created objectives (typed manually in the UI) correctly do *not* auto-generate tasks — the player remains in control and can use the Generate Tasks button when desired.

This was broken for most thinking models that reason more than a few hundred tokens. Autonomous proposals already had a generous 4000.
